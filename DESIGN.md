# 蚂蚁化工 (AntChem) — 设计宪法

版本 1.0 | 2026-06-19 | 本文件是项目所有视觉与交互设计的唯一权威来源。
任何新场景、新 UI 组件、新动效必须在此文件中找到约束或显式申请例外。

---

## 一、设计哲学

**蚁族微观世界的像素戏剧。**

- 玩家是一只 24px 高的行军蚁。一片草叶是摩天大楼。一颗石子是山丘。
- 所有设计决策从"这个昆虫能看见/感受到什么"出发。
- 像素不是技术限制——像素是美学选择。每一个像素都值得被故意放置。
- 暖黄萤火虫光 vs 深棕洞穴暗影：这个对比是游戏的情绪基调。
- 物理搞笑——严肃的化学操作 + 荒谬的物理后果。屏幕该抖的时候就抖到看不清。

**铁律：不要可爱化。** 蚂蚁是认真的工程师，干的也是危险的事。像素风不等于萌化。

### 场景色温递进

| 场景 | 主暖色 | 主冷色 | 情绪 |
|------|--------|--------|------|
| 洞穴实验室 | `--firefly` | `--cave-soil` | 温暖、安全、专注 |
| 试验场 | `--firefly` | `--cave-stone` | 紧张、期待 |
| 第一章 泥土之下 | `--firefly-dim` | `--cave-soil` | 幽闭、工程感 |
| 第二章 蚁后命令 | `--surface-green` | `--pure-white`(自然光) | 开阔、清新 |
| 第三章 红火蚁入侵 | `--lava-orange` | `--cave-deep` | 危险、紧迫 |

### 色彩使用规则

1. 任何场景的主色调不能超过 3 种。
2. `--firefly` 是游戏品牌色——在所有 UI 中这是默认高亮色。不需要第二种强调色。
3. 渐变仅限于：萤火虫光晕（径向渐变，从 `--firefly` 到透明）、爆炸闪光（径向渐变，从 `--pure-white` 到透明）、洞穴暗角（线性渐变，画面四角向中心的暗度渐变）。
4. 任何新颜色必须先申请加入主色板，不得在场景中直接 hardcode 新颜色。

## 三、像素精灵规格

### 基础单位

- **像素精度:** 32x32 per tile。所有 sprite 以 32px 为基准网格。
- **精灵尺寸范围:**
  - 蚂蚁角色: 24x20 px（含触角）
  - 原料图标: 16x16 px（库存/合成台缩略图）
  - 设备图标: 32x32 px（设备商店）
  - 建筑块 (RigidBody2D): 32x16 px（砖块）/ 32x32 px（方块）/ 64x32 px（大块岩石）
  - UI 按钮: 最少 64x32 px（2 tile 宽 × 1 tile 高），优先 96x32 px
  - 粒子: 4x4 px 或 8x8 px（单像素点可接受）

### 像素艺术铁律

1. **绝不使用亚像素坐标。** 所有 sprite position 必须 snap 到整数像素。Godot: `ProjectSettings > Rendering/2D/Snap > Snap 2D Transforms to Pixel = On`。
2. **绝不在像素精灵上使用旋转/缩放。** 如需旋转方块碎片——RigidBody2D 可以旋转，但精灵本身不旋转。碎片动画用多帧精灵表替代实时旋转。
3. **绝不使用抗锯齿纹理过滤器。** Godot: 所有 sprite 的 `Texture > Filter = Nearest`。
4. **轮廓线 1px 纯黑 (`--pure-black`)。** 所有前景元素（角色、原料、设备、可交互物体）必须有 1px 纯黑外轮廓。背景 tile 和环境元素不需要轮廓。
5. **光源来自单一方向（左上 45°）。** 所有 sprite 的高光和阴影遵循同一个光源假设：萤火虫灯在左上方。高光在左上边缘，阴影在右下边缘。

### 动画规格

| 动画类型 | 帧率 | 帧数 | 循环方式 |
|---------|------|------|---------|
| 蚂蚁待机 (idle) | 8 fps | 4 帧 | 循环 (呼吸式微动) |
| 蚂蚁行走 | 10 fps | 6 帧 | 循环 |
| 坩埚气泡/冒烟 | 6 fps | 8 帧 | 单次 (合成过程) |
| 爆炸闪光 | 15 fps | 4 帧 | 单次 |
| 碎片飞溅 | — | — | RigidBody2D 物理模拟，非精灵动画 |
| 萤火虫光摇曳 | 4 fps | 6 帧 | 循环 (亮度脉冲) |
| 合成图节点"点亮" | — | 1 帧渐变 | 灰色→暖黄 的 color modulate |
| UI 按钮 hover | — | — | 边框颜色 `--firefly-dim`→`--firefly` |

## 四、字体

### 像素字体选型

**主字体:** 需要一个 8px 或 16px 的像素字体（.ttf 或 bitmap font），满足以下条件：
- 包含 ASCII + CJK 基本字集（简体中文 6763 字）
- 等宽 (monospace)，匹配像素网格
- 候选: "fusion-pixel" (融合像素体, 开源, 含中文), "Zpix" (开源, 含中文), "最像素" (商业, 含中文)

若无合适像素中文字体，备选方案：
- ASCII/数字用像素英文字体 (如 "m3x6" 或 "Mini Font")
- 中文字回退到无衬线点阵字体

### 字号层级

| Token | 字号 | 行高 | 用途 |
|-------|------|------|------|
| `--text-title` | 16px | 1.2 | 场景标题、对话标题 |
| `--text-body` | 12px | 1.4 | 合成提示、设备描述、关卡目标 |
| `--text-small` | 10px | 1.3 | 原料名、设备名、数值标注 |
| `--text-tiny` | 8px | 1.2 | 合成图节点小字、HUD 状态 |

### 文字渲染

- Godot: 所有 Label 节点的 `Font > Oversampling = 0`（禁用抗锯齿渲染，保持像素锐利）
- 文字颜色: `--pure-white` 当在深色背景上；`--cave-deep` 当在 `--firefly` 背景上
- 禁止: 文字阴影、文字描边、文字渐变色

## 五、场景视觉规范

### 5.1 洞穴实验室

```
场景层级 (由远及近):
  1. 洞穴壁 (tilemap, 深棕 --cave-soil, 无轮廓)
  2. 工作台石板 (32x96 px, --cave-stone, 有轮廓)
  3. 设备 (32x32 px 图标, 已购买=暖黄光/未购买=灰色剪影)
  4. 原料散落 (16x16 px, 散落在工作台上)
  5. 坩埚 (32x32 px, 坚果壳纹理, 合成时冒泡/变色动画)
  6. 合成图面板 (左侧 280px 宽, 半透明深色背景, DAG 节点)
  7. 萤火虫 (右上角, 20x20 px 径向光晕, 4fps 亮度脉冲)

光环境: 单一光源-萤火虫光 (`--firefly`, 从右上角辐射)
暗角: 画面四角 15% 暗度渐变 (`--cave-deep` → transparent)
```

### 5.2 试验场

```
场景层级:
  1. 洞穴壁 (同实验室 tilemap)
  2. 地面 (底部 60px 高, 泥土 --cave-soil)
  3. 目标建筑 (RigidBody2D 块, PinJoint2D 连接)
     - 泥土块: --cave-soil
     - 石块: --cave-stone
     - 铁块 (第二章): --rust-red
  4. 炸药 (放置后可见, 8x12 px, --firefly 闪烁)
  5. 引线 (2px 宽, --firefly-dim, 点燃后从左到右燃烧)

引爆后:
  6. 闪光覆盖 (全屏 3 帧白色 flash → 0.3 秒消失)
  7. 碎片 (RigidBody2D, ≤40 块)
  8. 烟雾粒子 (GPUParticles2D, --smoke-gray, ≤150 个)
  9. 火焰粒子 (GPUParticles2D, --lava-orange, ≤100 个, 仅可燃炸药)
  10. 慢动作回放条 (底部, 半透明深色, 0.25x 速度标记)

UI 覆盖层:
  破坏率 (右上角, --text-title, --firefly)
  控制按钮 (左上角: [⏪慢放] [🔄重置] [📷导出GIF])
```

### 5.3 关卡场景通用规则

- 每个关卡场景有**一个退出按钮** (左上角, 64x32 px, "--firefly-dim 边框, 文字 '← 返回实验室'")。
- 关卡目标文字显示在顶部中央 (16px, `--text-title`, `--firefly`)，进入关卡后 2 秒出现。
- 关卡完成/失败时覆盖半透明暗幕 (`--cave-deep` 75% opacity)，文字居中显示结果和评分。
- 炸药背囊: 左下角 48x48 px 按钮，点击展开已制作炸药列表 (垂直列表，每个条目显示玩家命名的名字 + 16x16 图标)。

### 5.4 第一章 — 泥土之下

```
场景特征:
  - 封闭洞穴空间，四周被 tilemap 岩壁包围
  - 目标: 1-3 块大型硬岩 (64x32 px, --cave-stone) + 支撑柱 (3 块竖排, --cave-clay)
  - 隧道路径用虚线标记 (--firefly-dim, 2px 虚线, 从洞穴左侧通向右侧)
  - 蚂蚁工蚁在隧道入口等待 (24x20 px, idle 动画)

灯光: 萤火虫光 + 矿灯 (暖黄, 放在隧道入口)
```

### 5.5 第二章 — 蚁后的命令

```
场景特征:
  - 开放地表空间，草绿色调 (--surface-green 为主)
  - 天空: --pure-white 渐变到淡蓝 (#87CEEB, 仅此场景可用)
  - 草叶: 2px 宽竖线, 随机高度 40-80px, --surface-green, 轻微摇摆 (2fps 左右摇摆)
  - 障碍物:
    - 瓶盖: 60x20 px, 红色 --rust-red 圆形, 金属材质 (需高爆穿甲)
    - 火柴盒: 40x24 px, 米色纸板, 多层结构
    - 草茎: 4px 宽 × 60-100px 高, 纤维结构 (需节点爆破)
    - 碎石: 16-24 px 不规则方块, --cave-stone

灯光: 自然光 (全场景均匀亮度, 无单一光源)
```

### 5.6 第三章 — 红火蚁入侵

```
场景特征:
  - 敌巢前哨 (红色调, --rust-red 和 --lava-orange)
  - 外层墙壁: 3 层 RigidBody2D 堆叠, --rust-red
  - 蚁酸罐: 16x24 px, --acid-green 半透明容器, 爆炸后释放酸雾云 (GPUParticles2D, 绿色)
  - 幼虫库: 24x24 px, 米色半透明膜, 过早炸→幼虫逃散 (小型红色粒子向外扩散→增援信号)
  - 火蚁士兵: 24x20 px, --lava-orange 色, 在巢穴内巡逻 (来回行走动画)
  - 杀伤计数: 右上角 HUD, "🔥 ×3"

危险区域标记: 蚁酸罐周围 3 tile 半径画虚线 (--acid-green, 提示玩家避开)
```

## 八、粒子效果标准

### 通用规则

- **引擎:** GPUParticles2D（非 CPUParticles2D），Godot 4.x 原生支持
- **粒子数硬上限:** 全场景同时 ≤150 个活跃粒子
- **爆炸场景降级策略:** 如果当前帧率 < 30fps，自动将粒子数减半

### 烟雾

```
纹理: 8x8 px 软圆 (像素化圆形)
颜色: --smoke-gray, alpha 从 0.6 → 0
生命周期: 2-5s (取决于炸药)
发射形状: 球体 (爆炸半径的 0.8 倍)
初始速度: 50-150 px/s (向外), 带随机扰动
重力: 轻微向上 (-10 px/s², 热空气上升)
```

### 火焰

```
纹理: 8x8 px 不规则火焰形状
颜色: --lava-orange → --firefly → --smoke-gray (生命周期渐变)
生命周期: 1-3s
发射形状: 爆炸半径的 0.5 倍球体
初始速度: 80-200 px/s (向外+向上)
数量: 只对 fire_enabled=true 的炸药生成
```

### 碎片灰尘

```
纹理: 2x2 px 或 4x4 px 实心方块
颜色: --smoke-gray 或 --cave-clay
生命周期: 0.5-1.5s
初速度: 与 RigidBody2D 碎片同步, 带随机扰动
重力: 200 px/s² (向下掉落)
仅在 RigidBody2D 碎片 ≤40 块后剩余的微小块（<16px）转为粒子灰尘
```

## 九、音效策略 (Alpha 阶段实施)

- **原型阶段:** 无音效。OBS 录屏的 GIF 没有声音。
- **Alpha 阶段:** 用 jsfxr / SFXR 合成像素风音效（8-bit 风格, 匹配像素视觉）。
- **音效事件表:**

| 事件 | 类型 | 描述 |
|------|------|------|
| 合成成功 | 正面 | 短促上升音 (C-E-G 三音) |
| 合成失败 | 负面 | 低沉闷响 + 嘶嘶声 |
| 爆炸 (小型) | SFX | 短爆裂声, 0.2s |
| 爆炸 (大型) | SFX | 低频轰鸣 + 碎片散射声, 0.5-1s |
| 命名确认 | UI | 轻柔点击声 |
| 关卡完成 | 正面 | 胜利短曲 (3 音上升) |
| 关卡失败 | 负面 | 低沉下降音 |
| 设备购买 | UI | 金币声 (短促) |
| 萤火虫环境 | 环境 | 极轻微嗡鸣 (loop) |

- **Steam 阶段:** 评估独立音效师定制。

## 十、质量检查清单

任何新的视觉资产或 UI 变更在合并前必须通过以下检查：

- [ ] 所有 sprite 使用 32px 网格对齐，无亚像素坐标
- [ ] 所有纹理 Filter = Nearest（无抗锯齿）
- [ ] 所有前景元素有 1px `--pure-black` 轮廓线
- [ ] 高光/阴影假设光源为左上 45°
- [ ] 颜色限于主色板内的 token
- [ ] 按钮 ≥ 64x32 px 触控目标
- [ ] 合成图节点区分：已发现(暖黄方形)/中间体(绿色菱形)/未发现(灰色虚线方形)

## 十一、严禁事项 (Anti-Patterns)

1. **禁止亚像素坐标。** snap to pixel = ON, 永远。
2. **禁止精灵旋转/缩放。** 碎片物理旋转是 Godot RigidBody2D 的自动行为，不接受在此基础上叠加精灵旋转。
3. **禁止默认字体。** 系统默认字体 (Arial/Inter/Roboto) 在像素游戏中是视觉污染。必须使用选定的像素字体。
4. **禁止渐变色按钮/UI 元素。** 像素风 = 纯色块 + 纯色边框。唯一的渐变例外是光晕（萤火虫/爆炸闪光/暗角）。
5. **禁止 emoji 作为装饰元素。** ⚗/★/🔒 是功能性标记，不是装饰。禁止在 UI 中撒火箭/星星/火花 emoji。
6. **禁止文字描边/阴影。** 像素字体不需要这些。
7. **禁止半透明叠加层上的文字。** overlay 必须有足够的不透明度 (≥75%) 保证文字可读。
8. **禁止超过 3 种颜色在一个 UI 组件内。** 像素 UI = 约束之美。
9. **禁止新增颜色而不更新 DESIGN.md。**
10. **禁止关卡之间共享完全相同的视觉布局。** 每个关卡至少要有 1 个视觉差异化元素（岩石形状/障碍物布局/颜色强调）。
