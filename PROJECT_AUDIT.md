# 蚂蚁化工 (AntChem) — Steam 上架级别完整审核报告

> **审核日期:** 2026-06-24  
> **审核标准:** Steam 独立游戏发布品质  
> **项目版本:** 0.1.0-prototype  
> **引擎:** Godot 4.4  
> **结论: 当前完成度约 25%，距离可发布状态仍需大量工作**

## 零、总体评价

### 核心优势

- **设计文档 (DESIGN.md) 品质极高** — 18KB 的完整设计宪法，13 色板规范、场景层级、动效参数、像素艺术铁律、UI 组件库、反模式清单全部齐备。这是一个专业设计师产出的文档
- **核心循环概念正确** — 采集 → 合成 → 命名 → 爆破 → 关卡，这个循环有足够的深度和趣味性
- **程序化美术管线** — 51KB 的 generate_all.py + pixel_art 引擎，可以在不依赖外部美术的情况下迭代
- **配方系统有设计深度** — 容差匹配、操作锁、分支路线、爆炸效果差异化
- **代码架构清晰** — 3 个 Autoload 职责分明，场景脚本-场景文件分离，素材路径映射独立

### 致命缺陷

- **DESIGN.md 设计规范 0% 执行** — 13 个色彩 token 在代码中使用次数为 0，47 处 hardcode Color()
- **UI 完全依赖 Godot 默认主题** — 生成了像素按钮精灵但从未使用，所有按钮是灰色系统默认样式
- **Game Feel 为零** — 无场景过渡、无缓动动画、无 hover 反馈、无音效
- **内容严重不足** — 3 个章节关卡完全空白，试验场只有 1 种建筑配置
- **美术品质不达标** — 蚂蚁角色精灵 280 字节 (几乎没有像素细节)，多数精灵文件极小

## 四、交互与 Game Feel 审计

### 4.1 手感要素清单 (全部缺失)

| 要素 | 状态 | Steam 品质标准 |
|------|------|---------------|
| 场景过渡 (淡入淡出/黑屏) | ❌ | 必须 |
| UI 出现/消失动画 (Tween) | ❌ | 应该 |
| 按钮 hover 视觉反馈 | ❌ | 必须 |
| 按钮按下视觉反馈 | ❌ | 必须 |
| 合成成功庆祝效果 | ⚠️ 仅文字 | 应该有粒子/闪光 |
| 合成失败震动/反馈 | ❌ | 应该 |
| 炸药引爆倒计时感 | ❌ | 应该有引线燃烧+音效 |
| 摄像震动衰减 | ⚠️ while循环实现 | 应该用 Tween |
| 破坏率计数动画 | ❌ | 数字应该递增 |
| 萤火虫环境光动画 | ✅ sin() 脉冲 | 合格 |

### 4.2 场景过渡

当前所有场景切换使用 `get_tree().change_scene_to_file()`，硬切无过渡。Steam 品质要求至少:
- 0.3s 黑屏淡出 → 加载 → 0.3s 淡入
- 或像素风"帘幕关闭"效果 (从中间向上下展开黑条)

### 4.3 摄像震动实现问题

`test_field.gd` 的 `_apply_camera_shake` 使用 `while` + `await get_tree().process_frame` 实现:
- 在不同帧率下震动频率不一致
- 如果帧率低于震动频率，震动会卡顿
- 正确做法: 使用 `Tween` 或 `Camera2D` 的 noise 系统
